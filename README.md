# MODOC

Diagnose. Repair. Ready.

Find hidden problems in your 3D models.

MODOC analyses 3D models, detects geometry problems, repairs safe defects and confirms whether the model is ready for gaming, visualisation, 3D printing or manufacturing workflows. AI is used only to explain deterministic evidence; it does not invent geometry defects.

## Repository

```text
apps/api                 FastAPI geometry source of truth
apps/web                 Next.js inspection workspace
packages/shared-types    Shared TypeScript contracts
packages/ui              Small shared UI primitives
workers/geometry-analysis Worker placeholder for queue extraction
sample-data              Generated demo model provenance and fixtures
docs                     Architecture, scoring, safety and operations docs
tests                    End-to-end notes
```

## Install

```powershell
python -m venv .venv
.\.venv\Scripts\python -m pip install -r apps\api\requirements.txt
npm.cmd install
```

## Configure

```powershell
Copy-Item .env.example .env
```

Important variables:

```text
MODELDOCTOR_MAX_UPLOAD_MB=250
MODELDOCTOR_STORAGE_DIR=.modeldoctor_storage
MODELDOCTOR_MAX_TRIANGLES=2500000
MODELDOCTOR_MAX_REPAIR_TRIANGLES=750000
MODELDOCTOR_ALLOW_STEP=false
MODELDOCTOR_PREVIEW_TRIANGLE_LIMIT=120000
NEXT_PUBLIC_MAX_MODEL_UPLOAD_MB=250
OPENAI_API_KEY=
OPENAI_MODEL=gpt-4.1-mini
NEXT_PUBLIC_API_BASE=http://127.0.0.1:8010
```

## Start API

```powershell
.\.venv\Scripts\python -m uvicorn modeldoctor_api.main:app --reload --reload-dir apps\api --app-dir apps\api --host 127.0.0.1 --port 8010
```

API docs are available at `http://127.0.0.1:8010/docs`.

## Start Frontend

```powershell
$env:NEXT_PUBLIC_API_BASE="http://127.0.0.1:8010"
npm.cmd --workspace apps/web run dev -- --port 3001
```

Open `http://127.0.0.1:3001`.

## Load Demo Data

Demo STL files are generated by the API on startup and also by:

```powershell
.\.venv\Scripts\python apps\api\scripts\generate_demo_models.py
```

Use the **Load Demo Model** button in the web app.

## Current MVP

Implemented:

- ASCII and binary STL parsing.
- Basic OBJ parsing.
- Optional GLB parsing through `trimesh` when available.
- Deterministic metrics, duplicate vertices, duplicate faces, degenerate triangles, boundary edges, non-manifold edges, disconnected shells, tiny components, suspicious scale and far-origin checks.
- Health scoring by intended use.
- Safe repairs for duplicate vertices, duplicate faces and degenerate triangles.
- Repaired STL export, JSON issue export, CSV issue export and PDF report export.
- Built-in generated demo models.

Unsupported in this slice:

- STEP/STP B-rep inspection.
- Native proprietary CAD files.
- Full manufacturing validation, slicing simulation, FEA or machine-specific certification.
